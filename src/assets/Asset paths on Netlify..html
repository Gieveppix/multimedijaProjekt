The issue you're facing is likely due to incorrect paths for your assets in the deployed version of your app on Netlify. When deploying a Vue + Vite project on Netlify, the base URL may be different from the one in the development environment.

To fix the issue, you can use Vite's import.meta.env.BASE_URL to make sure you're using the correct base URL for your assets in both development and production environments.

First, make sure to add the following line in your index.html:

html
<base href="/" />

Then, you can update your asset paths in your Vue component using import.meta.env.BASE_URL. For example, you can update your audio path as follows:

javascript
onMounted(() => {
  stjepanAudio.value = new Audio(`${import.meta.env.BASE_URL}src/assets/audio-proba.mp3`);
});

You should do the same for all other asset paths in your component, such as images and videos. Here's an example for the VideoPlayer component:

html
<VideoPlayer videoUrl={`${import.meta.env.BASE_URL}src/assets/wr.mp4`} />

After updating all the asset paths, redeploy your app on Netlify, and it should work as expected.

I apologize for the confusion. In Vue templates, you cannot use JavaScript template literals directly. Instead, you can pass the URL as a computed property.

First, create a computed property for each video URL in your <script> section:

javascript
import { computed } from 'vue';

// Add this to your existing code
const wrVideoUrl = computed(() => `${import.meta.env.BASE_URL}src/assets/wr.mp4`);
const video1Url = computed(() => `${import.meta.env.BASE_URL}src/assets/video1.mp4`);
const video3Url = computed(() => `${import.meta.env.BASE_URL}src/assets/video3.mp4`);

Then, in your template, you can use the computed properties like this:

html
<VideoPlayer :videoUrl="wrVideoUrl" />
<VideoPlayer :videoUrl="video1Url" />
<VideoPlayer :videoUrl="video3Url" />

Make sure to use the colon : before the videoUrl attribute to bind the computed property correctly.
